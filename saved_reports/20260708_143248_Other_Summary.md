## 2. Per-Model Notes (harness engineering tips)

These small adjustments improve the honesty/comparability of each run without changing the
substance of the prompt:

- **ChatGPT:** Enable browsing/"Search" mode if available before pasting. Add nothing else — GPT
  tends to over-hedge less on the confidence column if you keep the table format strict, which is
  why the table (not prose) is the primary carrier of claims.
- **Gemini:** If using a version with Google Search grounding, confirm grounding is on; Gemini
  often silently blends grounded and ungrounded claims, so the "Search methodology" section at the
  end is especially important to inspect here.
- **Claude:** Web search is used automatically when available; check the "Search methodology"
  section it returns to confirm it actually searched rather than answered from training data alone.

Run all three within as short a time window as possible so market data (e.g. spot prices, recent
funding rounds) is comparable across runs.

## 4. Design notes — why this harness is structured this way

- **Role + context framing** anchors all three models to the same persona/expertise level, reducing
  variance caused by default generic-assistant tone.
- **Explicit track separation (A/B)** prevents models from conflating "cool lab technique" with
  "market-moving trend," a common failure mode in broad research prompts.
- **Table-first output format** forces atomic, comparable claims (rather than free-flowing prose),
  which is what makes automated/manual reconciliation in Part 3 possible.
- **Mandatory confidence + "could not verify" sections** counteracts each model's tendency to
  smooth over gaps with plausible-sounding filler — this is the main defense against silent
  hallucination during cross-checking.
- **Search methodology disclosure** exposes whether a model actually browsed live sources or is
  answering from a static knowledge cutoff, which materially changes how much you should trust its
  market-data rows.
- **The reconciliation prompt treats disagreement as signal, not noise** — corroboration across
  independently-run models is a cheap, practical proxy for reliability, while single-source claims
  are quarantined rather than silently promoted to "fact."
